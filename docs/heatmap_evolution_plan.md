# Training Heatmap → Epigenetic Evolution — Plan & Progress

**Status doc for handover between sessions.** Update the checklist and
"Current state" as layers land. Cite commits by hash.

## Goal

A heatmap is produced and **updated incrementally during training** so we
can **identify and mutate hot paths** using epigenetics, the formal
framework, and Lean. Decisions locked with the user:

- **Heat signal:** gradient magnitude (per-element `||∂loss/∂θ||`).
- **Granularity:** both **nodes** (modules/genes) and **edges** (paths),
  keyed by `HypergraphIR` element id.
- **Artifact:** incremental **JSON heatmap**; the **NFG compiler renders
  it when a `--heat` flag is supplied** (heat overlay on the graph).
- **Pipeline scope:** full — heatmap → propose `DNAPatch` mutations →
  gate via `ImprovementGate`/`TripleGuard` → **real `LeanProofBackend`**
  (no stub; shells out to `lean --json`).

## Architecture (pipeline)

```
training loop (harness)
  → per-param grads (named_parameters)         [L2 collector]
  → map param-name → IR element id (node/edge) [L2 mapper]
  → TrainingHeatmap.update(signals) every N    [L1 ✓]  → results/<arch>.heatmap.json
        │
        ├─ hot_paths()/cold_paths()            [L1 ✓]
        ▼
  propose DNAPatch mutations for hot nodes /   [L3]
  prune proposals for cold edges (epigenetics: neuroslm/compiler/epigenetics.py)
        ▼
  gate: ImprovementGate + TripleGuard          [L4]  (neuroslm/verification/)
        ▼
  LeanProofBackend: generate lean/Brian/*.lean,  [L5]
  run `lean --json`, parse, verdict
        ▼
  admitted mutations → apply to genome / DNA patches
```

NFG render overlay (`compile nfg --heat <heatmap.json>`) colors nodes/
edges by normalized heat.                                    [L6]

## Key existing building blocks (reuse, don't reinvent)

  `HyperEdge`, `lift_dsl_to_hypergraph`, `lift_arch_to_hypergraph`.
  Element ids: nodes `"<kind>:<name>"` (e.g. `population:gws`), edges
  `"<kind>:<src>-><dst>"` (e.g. `synapse:cortex->striatum`).

  `EvolutionContext`, `PatchComposer`; `DNAPatch` (in ribosome.py):

  `ImprovementGate.admit(before, after, *, direction)` →
  `ImprovementVerdict(admitted, effect, p_value, reasons, …)`;
  `CompositeGate`, `EvidenceTuple`.
- `neuroslm/verification/triple_guard.py` — structural admission
  (phi_min/h1_max/lambda_min).
- `docs/formal_framework.md` §10 specifies the Lean backend:
  `LeanProofBackend` shells out to `lean --json` against
  autogenerated `.lean` in `lean/Brian/`. Planned theorems:
  `PhiMonotone.lean`, `OodGapDecrease.lean`, `SymbolicSparsity.lean`,
  `TripleGuardSound.lean`.

  (activation_corr/firing_rate were illustrative; **we use grad-norm**).
- Harness gradient access: `neuroslm/harness.py`, iterate
  `model.named_parameters()` (see lines ~81/119/128).
