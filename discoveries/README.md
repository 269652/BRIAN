# `discoveries/` — autodiscovered mutation ledger

The engine-authored complement to [`../hypothesis/`](../hypothesis/).
Every mutation the evolutionary loop generates *and* the admission
gates accept (`TripleGuard` + `ImprovementGate`, see
[`docs/formal_framework.md`](../docs/formal_framework.md) §§6.4, 9) is
appended here as a `DiscoveryRecord`. Once the sibling Lean proof
under `proofs/` verifies, the discovery can be **spliced back into the
DNA** via [`splice_discovery_into_dna`](../neuroslm/discoveries/splice.py).

The whole flow:

```
EvolutionaryEngine ──► mutation ──► TripleGuard ──► ImprovementGate
                                                         │
                                                         ▼
                                               DiscoveryRecord (proof_status: missing)
                                                         │
                                                         ▼
                                      emit_discovery_proof  ──►  proofs/D00N_*.lean
                                                         │
                                                         ▼
                                       verify_lean_proof  ──►  proof_status: verified
                                                         │
                                                         ▼
                                       splice_discovery_into_dna  ──►  arch.neuro
                                                         │
                                                         ▼
                                                  dna_integrated: true
```

## File layout

```
discoveries/
  README.md                          ← this file
  schema.json                        ← JSON Schema for DiscoveryRecord
  index.json                         ← autogenerated index (don't hand-edit)
  D001_*.md                          ← one .md per admitted mutation
  D002_*.md
  ...
  proofs/
    D001_*.lean                      ← one .lean per discovery
    ...
```

## Schema

Authoritative dataclass:
[`neuroslm.discoveries.records.DiscoveryRecord`](../neuroslm/discoveries/records.py):

| Field                   | Type                | Meaning |
|-------------------------|---------------------|---------|
| `id`                    | `D\d{3,}`           | Stable identifier (e.g. `D001`) |
| `title`                 | `str`               | One-line description |
| `mechanism_md`          | `str` (body)        | Markdown body — what the mutation does |
| `mutation_chain`        | `list[str]`         | Names of mutation ops applied (e.g. `["add_modulation", "mutate_numeric"]`) |
| `mutation_args_json`    | `str` (JSON list)   | Arguments for each op — used by the splice |
| `parent_dna_id`         | `str`               | Parent genome (DNA hash, checkpoint name, …) |
| `fitness_before`        | `dict[str, float]`  | Metrics before the mutation |
| `fitness_after`         | `dict[str, float]`  | Metrics after the mutation |
| `fitness_delta`         | `dict[str, float]`  | Auto-computed (`after - before`) |
| `generation`            | `int`               | Generation number in the evolutionary run |
| `theorem_name`          | `str`               | Lean obligation name (e.g. `Brian.Discoveries.D001_NoRegression`) |
| `proof_path`            | `str` or `null`     | Path to the `.lean` stub / proof |
| `proof_status`          | enum                | `missing` ╱ `stub` ╱ `compiles` ╱ `verified` |
| `dna_integrated`        | `bool`              | True iff the mutation has been spliced into the genome |
| `dna_integrated_at`     | ISO-8601 or `null`  | When the splice happened |
| `hypergraph_delta_json` | `str` (JSON)        | Diff against the parent IR (added/removed nodes/edges) |
| `tags`                  | `list[str]`         | Free-form filters |

## How a discovery enters the ledger

The hook `neuroslm.discoveries.hooks.record_admitted_mutation` (called
by the evolutionary loop after both admission gates pass) does:

1. Build a `DiscoveryRecord` from the engine's state.
2. `DiscoveryStore.save()` writes the `.md`.
3. `emit_discovery_proof()` writes the `.lean` stub.
4. (optional) `verify_lean_proof()` updates `proof_status`.
5. If verified, `splice_discovery_into_dna(arch_root)` promotes the
   mutation; the file's `dna_integrated` bit flips to `true`.

Every step is auditable: the `.md` files are git-friendly, `.lean`
files are mechanically checkable, and the splice block in `arch.neuro`
carries a comment header with the discovery id.

## The verification firewall

A discovery's `dna_integrated` bit can only be set when
`proof_status == "verified"`. There is no override. This is the
firewall: the engine can propose any mutation, but the genome only
accepts mutations whose no-regression obligation has been formally
discharged.

Without the Lean binary on `$PATH` no discovery can be promoted —
that's intentional. The matched outcome (`proof_status="stub"` stays
forever) is preferable to silently widening the trust boundary.

## Examples

This folder ships empty (or with a single seeded example) on purpose
— the engine populates it during evolutionary runs. To kick a run
locally:

```bash
brian train --dna architectures/rcc_bowtie/evolution.dna --steps 1000
# admitted mutations land in discoveries/ automatically
```
